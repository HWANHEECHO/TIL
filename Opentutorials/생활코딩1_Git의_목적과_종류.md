

# Git의 목적

## (1) 버전관리(Version)

`commit`으로 디렉토리 내 파일들의 작업 후 버전에 내용과 일자를 포함하여 효과적 관리 가능



## (2) 백업(Backup)

`push`를 이용하여 `(remote) repository`에 백업 가능

`pull`을 이용하여 `repository`에서 버전 찾아오기 가능



## (3) 협업(Collaborate)

`push`와 `pull`을 타인과 함께하며 협업 가능

동일한 파일을 동시에 수정한 경우에 대한 대처 기능도 포함
